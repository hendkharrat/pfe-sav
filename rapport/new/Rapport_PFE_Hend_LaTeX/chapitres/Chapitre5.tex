\section{Introduction}

Suite à l'implémentation des fonctionnalités d'authentification, de gestion du référentiel métier et
de gestion des interventions lors des sprints précédents, ce troisième sprint se concentre sur les
fonctionnalités de supervision et de suivi général du système.

Ce sprint a pour objectif de compléter l'application en y intégrant :

\begin{itemize}
    \item La génération des factures pour les interventions curatives hors contrat ;
    \item Le marquage des factures comme payées ;
    \item L'historique des interventions, différencié par rôle ;
    \item Les tableaux de bord différenciés (administrateur, technicien, client).
\end{itemize}

\subsubsection{CU-16 : Générer une facture}

\begin{table}[H]
\centering

\begin{tabular}{|p{3.5cm}|p{11cm}|}
\hline
\textbf{Champ} & \textbf{Contenu} \\
\hline
Description & Ce cas d'utilisation permet à l'administrateur de générer une facture pour une
intervention éligible. \\
\hline
Acteur principal & Administrateur \\
\hline
Préconditions & L'administrateur est authentifié ; une intervention curative réalisée et hors
couverture contractuelle existe. \\
\hline
Postconditions & La facture est créée et enregistrée dans la base de données. \\
\hline
\end{tabular}
\end{table}

\textbf{Règles d'éligibilité (trois conditions cumulatives) :}

\begin{itemize}
    \item Type de l'intervention = \texttt{CURATIVE}
    \item Statut de l'intervention = \texttt{REALISEE}
    \item Couverture contractuelle = false (\texttt{couvertureContrat} = false)
\end{itemize}

\textbf{Scénario nominal :}

\begin{enumerate}
    \item L'administrateur accède à la liste des interventions éligibles à la facturation.
    \item Il sélectionne une intervention répondant aux trois conditions.
    \item Il saisit les lignes de détail de la facture (\texttt{LigneFacture}) : main-d'œuvre et matériel utilisé.
    \item L'Interface transmet les données au Contrôleur.
    \item Le Contrôleur vérifie les trois conditions d'éligibilité.
    \item Le Contrôleur calcule le montant HT (somme des lignes), la TVA (19\,\%) et le montant TTC.
    \item La base de données enregistre la facture et ses lignes de détail.
    \item La base confirme la création.
    \item L'Interface affiche la facture générée.
\end{enumerate}

\textbf{Scénario alternatif --- Intervention non éligible :}

\begin{itemize}
    \item L'une des trois conditions n'est pas satisfaite (intervention préventive, ou non réalisée,
          ou couverte par contrat).
    \item Le Contrôleur retourne une erreur.
    \item L'Interface affiche « Intervention non éligible à la facturation ».
\end{itemize}

\textbf{Règles de gestion :}

\begin{itemize}
    \item \textbf{Les interventions préventives ne génèrent jamais de facture.}
    \item \textbf{Les interventions curatives couvertes par un contrat actif ne génèrent pas de facture.}
    \item La TVA est de \textbf{19\,\%} : \texttt{montantTTC} = \texttt{montantHT} $\times$ 1,19.
    \item La monnaie est le \textbf{Dinar Tunisien (TND)}.
    \item La facture est créée avec le statut \texttt{IMPAYEE} ou \texttt{EN\_ATTENTE}.
\end{itemize}

\subsubsection{b) Identification textuelle --- Consultation des factures}

\textbf{Nom du cas d'utilisation :} Consulter les factures \quad
\textbf{Acteur principal :} Administrateur / Client

\textbf{Objectif :} Permettre à l'utilisateur de consulter les factures enregistrées dans le système
selon ses droits d'accès.

\textbf{Préconditions :}

\begin{itemize}
    \item L'utilisateur est authentifié.
    \item Des factures existent dans la base de données.
\end{itemize}

\textbf{Postconditions :}

\begin{itemize}
    \item Les factures accessibles à l'utilisateur sont affichées en lecture seule.
\end{itemize}

\textbf{Scénario nominal :}

\begin{enumerate}
    \item L'utilisateur ouvre le module des factures.
    \item L'Interface transmet la demande au Contrôleur avec l'identifiant et le rôle de l'utilisateur.
    \item Le Contrôleur vérifie le rôle :
    \begin{itemize}
        \item Si rôle = admin $\rightarrow$ la base de données retourne toutes les factures.
        \item Si rôle = client $\rightarrow$ la base de données retourne uniquement les factures dont
              le \texttt{clientId} correspond à l'utilisateur connecté.
    \end{itemize}
    \item La base de données retourne les enregistrements.
    \item Le Contrôleur retourne les données à l'Interface.
    \item L'Interface affiche les factures avec les filtres et la pagination.
\end{enumerate}

\textbf{Règles de gestion :}

\begin{itemize}
    \item L'administrateur voit toutes les factures ; le client ne voit que les siennes.
    \item La consultation est en lecture seule pour le client.
    \item Des filtres sont disponibles : statut (\texttt{PAYEE}, \texttt{IMPAYEE}, \texttt{EN\_ATTENTE}),
          date d'émission.
\end{itemize}

\subsubsection{d) Identification textuelle --- Consultation de l'historique avec filtres}

\textbf{Nom du cas d'utilisation :} Consulter l'historique avec filtres \quad
\textbf{Acteur principal :} Administrateur / Technicien / Client

\textbf{Objectif :} Permettre à l'utilisateur de consulter l'historique des interventions selon son
rôle et des critères de filtrage.

\textbf{Préconditions :}

\begin{itemize}
    \item L'utilisateur est authentifié.
    \item Des interventions avec statut \texttt{REALISEE} ou \texttt{ANNULEE} existent dans le système.
\end{itemize}

\textbf{Postconditions :}

\begin{itemize}
    \item L'historique filtré selon le rôle est affiché avec pagination.
\end{itemize}

\textbf{Scénario nominal :}

\begin{enumerate}
    \item L'utilisateur ouvre le module Historique.
    \item L'Interface transmet la demande au Contrôleur avec l'identifiant, le rôle et les filtres.
    \item Le Contrôleur détermine le périmètre visible selon le rôle :
    \begin{itemize}
        \item Admin $\rightarrow$ toutes les interventions \texttt{REALISEE} ou \texttt{ANNULEE}.
        \item Technicien $\rightarrow$ ses interventions assignées avec ces statuts.
        \item Client $\rightarrow$ les interventions liées à ses \texttt{ClientEquipement}.
    \end{itemize}
    \item Le contrôleur interroge la base de données avec les filtres appropriés.
    \item La base retourne les enregistrements.
    \item L'Interface affiche les résultats avec pagination.
\end{enumerate}

\textbf{Règles de gestion :}

\begin{itemize}
    \item Seules les interventions avec statut \texttt{REALISEE} ou \texttt{ANNULEE} apparaissent
          dans l'historique.
    \item Le technicien ne voit que ses propres interventions.
    \item Le client ne voit que les interventions liées à ses équipements affectés.
\end{itemize}

\textbf{Objectif :} Permettre au technicien de consulter ses indicateurs personnels.

\textbf{Préconditions :}

\begin{itemize}
    \item Le technicien est authentifié.
\end{itemize}

\textbf{Postconditions :}

\begin{itemize}
    \item Le tableau de bord personnel du technicien est affiché.
\end{itemize}

\textbf{Scénario nominal :}

\begin{enumerate}
    \item Le technicien accède à son tableau de bord.
    \item L'Interface transmet la demande au Contrôleur avec l'identifiant du technicien.
    \item Le contrôleur récupère les interventions assignées au technicien depuis la base de données.
    \item Le Contrôleur calcule les indicateurs : nombre total d'interventions assignées, nombre en cours,
          nombre réalisées, taux de réalisation.
    \item Le Contrôleur récupère les interventions planifiées pour la semaine en cours.
    \item L'Interface affiche le tableau de bord.
\end{enumerate}

\textbf{Règles de gestion :}

\begin{itemize}
    \item Le tableau de bord est filtré sur le technicien connecté uniquement.
    \item Le taux de réalisation = interventions \texttt{REALISEE} / total interventions assignées
          $\times$ 100.
\end{itemize}

\section*{Conclusion}

Ce sprint a permis de finaliser le système SAV avec les fonctionnalités de pilotage : génération
des factures selon les règles d'éligibilité strictes (curative + réalisée + hors contrat), calcul
automatique de la TVA à 19\,\% en dinar tunisien, marquage des factures payées, et des tableaux de
bord différenciés pour l'administrateur, le technicien et le client.
