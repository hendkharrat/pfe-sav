Suite à l'implémentation des fonctionnalités fondamentales du système lors du Sprint 1, ce deuxième
sprint se concentre sur la gestion des interventions préventives et correctives, la déclaration des
pannes et le calendrier des techniciens. Ce sprint constitue le noyau fonctionnel du système SAV.

Les caractéristiques élaborées au cours de ce sprint portent principalement sur :

\begin{itemize}
    \item La vérification du calendrier sur une base hebdomadaire et mensuelle ;
    \item La répartition des techniciens avec contrôle de disponibilité ;
    \item Le début et la fin des interventions par les techniciens ;
    \item Les clients peuvent déclarer des pannes avec plusieurs pièces jointes ;
    \item La gestion et la transformation des pannes en actions correctives ;
    \item L'automatisation de la vérification de la couverture contractuelle lors de la clôture.
\end{itemize}

\textbf{Scénario nominal :}
\begin{enumerate}

    \item L'Interface transmet la demande au Contrôleur avec l'identifiant de l'intervention
          et du technicien.
    \item Le Contrôleur vérifie l'existence de l'intervention.
    \item Le Contrôleur vérifie que le technicien est actif.
    \item Le Contrôleur vérifie la disponibilité du technicien à la date prévue.
    \item La base de données confirme que le technicien n'a pas d'autre intervention
          à cette date.
    \item Le Contrôleur met à jour l'intervention avec le technicien affecté et le statut
          \texttt{PLANIFIEE}.
    \item La base de données effectue la mise à jour.
    \item L'Interface confirme l'affectation.
\end{enumerate}

\textbf{Scénarios alternatifs :}
\begin{itemize}
    \item SA1 : Intervention introuvable $\rightarrow$ message d'erreur.
    \item SA2 : Technicien inactif ou introuvable $\rightarrow$ message d'erreur.
    \item SA3 : Technicien indisponible à cette date $\rightarrow$ message
          « Technicien indisponible à la date prévue ».
    \item SA4 : Erreur base de données $\rightarrow$ message d'erreur.
\end{itemize}

\textbf{Règles de gestion :}
\begin{itemize}
    \item Seuls les techniciens actifs peuvent être affectés.
    \item \textbf{Un technicien ne peut pas être affecté à deux interventions prévues
          à la même date.}
    \item L'affectation met automatiquement le statut de l'intervention à \texttt{PLANIFIEE}.
\end{itemize}

\textbf{Scénario nominal :}
\begin{enumerate}
    \item Le client accède au module de déclaration de panne.
    \item Il sélectionne l'un de ses équipements affectés (parmi ses \texttt{ClientEquipement}).
    \item Il saisit une description de la panne (\textbf{obligatoire}).
    \item Il joint éventuellement une ou plusieurs pièces jointes (images, PDF, fichiers audio).
    \item Il soumet la déclaration.
    \item L'Interface transmet les données au Contrôleur.
    \item Le Contrôleur vérifie que l'équipement appartient bien au client.
    \item Le Contrôleur enregistre la panne avec ses \texttt{PieceJointe[]} associées.
    \item La panne est créée avec le statut \texttt{EN\_ATTENTE}.
\end{enumerate}

\textbf{Scénarios alternatifs :}
\begin{itemize}
    \item SA1 : Description manquante $\rightarrow$ validation échoue ;
          message « Description obligatoire ».

          « Équipement introuvable ».
\end{itemize}

\textbf{Règles de gestion :}
\begin{itemize}
    \item La description est obligatoire.
    \item \textbf{Aucun champ de priorité n'est présent.}
    \item Les pièces jointes sont de type image, PDF ou audio.
    \item La panne est créée avec le statut \texttt{EN\_ATTENTE}.
\end{itemize}

    \item Le Contrôleur vérifie les informations.
    \item Le \textbf{Système} vérifie automatiquement la couverture contractuelle pour cet
          enregistrement \texttt{ClientEquipement}.
    \item L'attribut \texttt{couvertureContrat} est renseigné automatiquement.
    \item L'administrateur sélectionne un technicien.
    \item Le Contrôleur vérifie la disponibilité du technicien à la date prévue.
    \item La base de données enregistre l'intervention curative avec le technicien affecté
          et le statut \texttt{PLANIFIEE}.
    \item La base confirme la création.
    \item L'Interface affiche une confirmation à l'administrateur.
\end{enumerate}

Scénario alternatif --- Technicien indisponible :
\begin{itemize}
    \item La base de données retourne que le technicien est indisponible à la date prévue.
    \item L'Interface affiche « Technicien indisponible à la date prévue ».
\end{itemize}

Règles de gestion :
\begin{itemize}
    \item \textbf{Aucun champ de priorité n'est associé à l'intervention curative.}
    \item La couverture contractuelle est vérifiée automatiquement par le Système.
    \item Seul un technicien actif et disponible peut être affecté.
\end{itemize}
